## 📋 Описание варианта
В рамках лабораторной работы спроектирована и реализована объектная модель для ресторанного бизнеса, включающая:

1. **Базовый класс 1 (Сущность):** `MenuItem` (обязательный атрибут: `price`)
2. **Дочерние классы 1:** 
   - `Dish` (добавляет атрибут: `ingredients`)
   - `Drink` (добавляет атрибут: `volume_ml`)
3. **Базовый класс 2 (Субъект):** `Customer` (обязательный атрибут: `table_number`)
4. **Дочерний класс 2:** `LoyaltyCardHolder` (добавляет атрибут: `points`)
5. **Класс-контейнер:** `Check` (агрегирует клиента и список позиций меню)

## ⚙️ Реализованная бизнес-логика (Полиморфизм)
Реализован полиморфный метод `apply_discount(is_loyal: bool) -> float`:
- В базовом классе `MenuItem`: возвращает `0.0` (базовая реализация, скидки нет).
- В классе `Dish`: возвращает `15%` от цены, если гость является держателом карты лояльности (`is_loyal == True`).
- В классе `Drink`: возвращает `0.0` (на напитки скидка не распространяется, даже для лояльных гостей).
